🚨
Incidente Grave: Perfis de Membros Foram Mesclados Automaticamente
Segurança de Dados Maio / 2026
O que aconteceu

O InChurch rodou uma rotina automática que fundiu perfis de membros que tinham o mesmo primeiro nome. Vários "Bruno" foram tratados como uma única pessoa — em menos de 3 segundos. O processo foi automático, sem aviso prévio.

Por que é grave

Dados pessoais sensíveis (CPF, RG, histórico) de pessoas diferentes podem ter sido misturados e expostos para quem não deveria ver. Isso é uma possível violação da LGPD (Lei Geral de Proteção de Dados).

O que o InChurch disse

A mesclagem foi desativada e os perfis foram restaurados. Protocolo de atendimento: 45143624195.

O que ainda não sabemos
  • ❓ Como o sistema garantiu que o CPF do "Bruno A" voltou para o "Bruno A" certo, se ambos tinham só o primeiro nome cadastrado?
  • ❓ Quantos perfis foram afetados no total? Não recebemos esse relatório de impacto.
Por que isso aconteceu — dado que confirma
O campo CPF está preenchido em apenas 21% dos participantes. Sem um identificador único, o sistema não tem como distinguir pessoas com nomes parecidos. Este não é um problema isolado — é uma falha estrutural no cadastro.

O InChurch tem um número único de membro — mas ele só existe no módulo de Membros. Ele não aparece em Participantes, nem em Líderes, nem em Supervisores. Problema do Reset
O que acontece quando os dados de participantes nunca são revisados ou limpos.

Quando uma pessoa para de frequentar uma célula mas não é removida do sistema, ela continua contando como participante. Com o tempo, os números ficam inflados e perdem o significado real. O número 6.376 pode incluir dezenas ou centenas de pessoas que não frequentam mais.

Como funciona o campo "Última Participação"

O campo "Última Participação" no sistema não é preenchido automaticamente com uma data. Ele registra o nome da célula mais a data da reunião, assim:

Fabrício e Andressa Moskwyn – 20/02/2026 – 20:00
É uma anotação de texto, não um campo de data — por isso o sistema não consegue filtrar automaticamente quem está inativo.

O que um "reset" significaria

Revisar periodicamente (por exemplo, uma vez por ano) se cada participante ainda frequenta a célula — e remover ou inativar quem não vai mais. Isso tornaria o número de participantes um reflexo real da realidade, permitindo planejamento mais preciso de liderança, espaço e recursos.

Não é uma crítica ao trabalho pastoral — é uma questão de qualidade de dados para tomada de decisão.
